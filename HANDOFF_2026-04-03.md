   `dirtdvlsa` does NOT appear in `Sega Model 3.xml`. It is not a LaunchBox game entry.
   That error was generated by the **`Sega Model 3 UI.exe` GUI frontend** being tested independently,
   not by the Arcade Assistant Python adapter. We chased this red herring for part of the session.

3. **`Sega Model 3 UI.exe`** is a separate GUI wrapper around `Supermodel.exe`.
   Its `[Supermodel3 UI]` section in `Supermodel.ini` has `Dir=` **BLANK**.

   This is irrelevant to our launch path IF we are correctly launching `Supermodel.exe` directly.

4. **Supermodel has NO global ROMPath** in its INI file.
   It requires a fully-qualified absolute path to the ROM ZIP on the command line.
   The emulator does NOT search any ROM directory on its own.

5. **The emulator ID** in LaunchBox for Sega Model 3 is `d881d2d5-81ff-4d66-a1c4-443f07c4a671`.
   We did NOT yet check what `ApplicationPath` this ID maps to in `Emulators.xml`.

## 4. The Most Probable Root Causes (Read These Carefully)

Listed in order of likelihood. Work through them top-to-bottom:

### 🔴 Cause A: LaunchBox Emulator Entry Points to `Sega Model 3 UI.exe`, Not `Supermodel.exe`

This is the most likely culprit. LaunchBox has its own launcher for Model 3 emulator `d881d2d5-...`.
If that entry's executable is the UI frontend, Python never gets invoked regardless of PLUGIN_BYPASS.

**How to check:**
```
File: W:\Arcade Assistant Master Build\LaunchBox\Data\Emulators.xml
Search for: d881d2d5-81ff-4d66-a1c4-443f07c4a671
Read the <ApplicationPath> inside that <Emulator> block
```

If it says `Sega Model 3 UI.exe` → LaunchBox is handing off to the UI, which then fails because `Dir=` is blank.

**Fix:** Either:
- Change the LaunchBox emulator entry to point directly to `Supermodel.exe`
- OR set `Dir=W:\Arcade Assistant Master Build\Roms\MODEL3` in the `[Supermodel3 UI]` section of `Supermodel.ini`

### 🔴 Cause B: The Adapter Is Never Being Called

Even with PLUGIN_BYPASS in place, the launch route might not reach `supermodel_adapter.py`.

**How to verify immediately:**
Add this debug line at the very top of `build_command()` in `supermodel_adapter.py`:
```python
import logging as _logging
_logging.getLogger(__name__).warning(f"[SUPERMODEL ADAPTER] build_command called. rom_path={rom_path}")
```
Then restart the backend, launch a Model 3 game from LoRa, and check the backend log output.
If you do NOT see this warning, the adapter is not being reached.

### 🔴 Cause C: ROM Files Don't Exist on W: Drive

The LaunchBox XML is clean, but the actual ZIP files may not be there.

**Quick check:**
```powershell
dir "W:\Arcade Assistant Master Build\Roms\MODEL3\"
```
Look for `scud.zip`, `vf3.zip`, `bass.zip` etc. If the folder is empty or missing, no amount of
path fixing will help — you need to populate the ROMs first.

### 🟡 Cause D: Supermodel CWD Not Set Correctly

Supermodel.exe needs to run with its own directory as the working directory. It looks for:
- `Config\Supermodel.ini` (relative to CWD)
- `NVRAM\` (relative to CWD)

Verify in `supermodel_adapter.py` that the adapter returns:
```python
cwd = str(exe.parent)  # W:\Arcade Assistant Master Build\Emulators\Super Model
```
